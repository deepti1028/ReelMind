Edge Case: Deleted Reel Referenced in Chat History
When a reel is deleted (deleted_at set), its reel_chunks rows are hard-deleted immediately, removing it
from the RAG pipeline entirely. However the reel row itself is soft-deleted so it remains referenceable. The
sources JSONB on existing chat_messages may contain the reel_id of this now-deleted reel. The iOS app
must handle this gracefully: when rendering a source card, check whether the referenced reel has
deleted_at set. If so, render a degraded card showing "This reel was deleted" instead of the thumbnail and
Watch Reel button. The assistant text response is unaffected — only the source card degrades. A full
solution (e.g. flagging affected messages, notifying the user) is deferred to a later pipeline as noted in the
PRD.
