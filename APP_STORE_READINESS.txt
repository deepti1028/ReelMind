2.2 AI Data Disclosure (Guideline 5.1.2 — Enforced November 2025)
------------------------------------------------------------------
STATUS: NOT IMPLEMENTED — BLOCKING.

ReelMind sends user data to external AI services on every reel save:
  - Groq Whisper (whisper-large-v3): receives the reel's audio (.m4a file)
  - Groq Llama 3.3 70B: receives transcript + caption + hashtags for classification
  - Google Gemini (gemini-embedding-2): receives transcript + caption + hashtags

None of this is disclosed to the user in a consent modal before it happens.
The iOS app shows no consent screen at any point in the flow — not during
onboarding, not at first save, not anywhere.

This is a hard rejection under Guideline 5.1.2 as of November 2025.

What you need:
  A modal (shown before the first reel is ever processed) that names each
  AI provider (Groq, Google Gemini) and explains what data is sent to each.
  User must actively accept before any reel is submitted to the backend.

Note: the processing happens on the backend (Celery worker), not in the iOS
app. But the iOS app initiates the flow, so the disclosure obligation falls
on the iOS app.
